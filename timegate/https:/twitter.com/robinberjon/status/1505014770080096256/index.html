Aram Zucker-Scharff (Chronotope): @robinberjon I'm very interested in how Github Actions (and the like) can make building rich indexes cheap and easy if handled properly. It's interesting to think how that might come together with an federation model and a Wikipedia editors approach to topical maintenance and interests.

Robin Berjon (robinberjon): @Chronotope I've been wondering about distributed indices but I don't know if it can be done with potentially adversarial participants?

Aram Zucker-Scharff (Chronotope): @robinberjon I think there's a good model in how Wikipedia editors handle refining and working on a piece and while, mechanically it is adversarial, I don't think it is philosophically. A mechanism for merging indices with particular priorities or outcome data seems like it would be useful.

Robin Berjon (robinberjon): @Chronotope Maybe I don't have a clear view of what you have in mind, but that's not the adversarial I have in mind. I'm worried about malicious actors deliberately decreasing the value. Indexing has to be automated, there's too much drudgery. How do you only get good indexing?

melody joy kramer (mkramer): @robinberjon @Chronotope You hire librarians.

Aram Zucker-Scharff (Chronotope): @mkramer @robinberjon Yeah, this too! It isn't accidental that Wikipedia editors and librarians tend to overlap. I think the thing we need is a way that empowers people with passion and expertise to make the best indexes and represent themselves as candidates to join your search tool to.

Robin Berjon (robinberjon): @Chronotope @mkramer I love this model (and librarians)! So, this would need shared access to crawl (so as not to duplicate infrastructure, à la common crawl), an API to query federated indices, and ILP so people can be paid and infra costs covered. It's not crazy hard!

Aram Zucker-Scharff (Chronotope): @robinberjon @mkramer I don't know if you'd need a shared crawler, crawling is easy and these days fairly cheap. You just need a shared crawler standard and an agreement to use a common user agent I think?

Aram Zucker-Scharff (Chronotope): @robinberjon @mkramer I think this could get really interesting if it came together with Hyperdrive, your browsing than stands in for the crawler and your storage hosts an index which gets auto joined via connections, with larger servers using Hyper agents to pull federation into the standard web.

Robin Berjon (robinberjon): @Chronotope @mkramer I considered that, but how do you prevent this from including super private stuff that shows up in page?

Aram Zucker-Scharff (Chronotope): @robinberjon @mkramer Yeah that part is a little messy, but maybe start with 'click to add', build a blocklist, evolve the process. Private URLs are a relatively small set compared to the rest of the web I'd bet, and many have common characteristics.

Robin Berjon (robinberjon): @Chronotope @mkramer I don't know, that would not be foolproof enough. Like, your name would be caught up with news content if you're signed in.

Robin Berjon (robinberjon): @Chronotope @mkramer How about: sites *push* their content to be indexed, along with IP terms (to prevent the hostile mining that Google does but also independent repub), and Hyper agents are used to verify that real site content matches the index, to catch cheaters.

Robin Berjon (robinberjon): @Chronotope @mkramer Plus, detection of interstitials, notification prompts, bad cookie banners, perf...
